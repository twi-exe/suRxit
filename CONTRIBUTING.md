## Branch Naming## Branch Naming

- Use `feature/<short-description>`, `bugfix/<short-description>`, or `docs/<short-description>`- Use `feature/<short-description>` for new features

- Example: `feature/ocr-service`- Use `bugfix/<short-description>` for bug fixes

- Use `docs/<short-description>` for documentation

## Commit Conventions

- Use [Conventional Commits](https://www.conventionalcommits.org/):## Commit Conventions

  - `feat: add new NER pipeline`- Use [Conventional Commits](https://www.conventionalcommits.org/):

  - `fix: correct entity linking bug`  - `feat: add new NER pipeline`

  - `docs: update README`  - `fix: correct KG entity mapping`

  - `docs: update README`

## Code Review Rules

- All PRs require at least 1 reviewer approval## Code Review Rules

- Ensure tests and linting pass before merging- All PRs require at least one approval

- Link issues in PR descriptions- Write clear PR descriptions referencing issues

- Use clear, descriptive PR titles- Add/maintain tests for new features

- Ensure CI passes before merging
